Building a cloud platform part 1: My first etcd cluster

Elijah Caine | Mon 17 July 2017
Resources:
https://github.com/coreos/etcd/blob/master/hack/tls-setup/Procfile https://coreos.com/etcd/docs/latest/op-guide/configuration.html https://github.com/coreos/coreos-vagrant/

No preambles, just tech and gifs.

Anywhozle, the fist step to deploying a Kubernetes cluster is to deploy an etcd cluster, [obvs], so let's do that.

Cool. That wasn't so bad. Now let's make it secure.

I was originally going to hack together some Terraform local shell execution to get local certs generated and deployed, but then I found the Terraform TLS provider. I was hoping for a LetsEncrypt provider, but having already gotten ankle deep down the "shell script execution in Terraform" rabbit hole, I took what I could get.

As much as I would like to show you alllllll of the intermittent failures I went through, including a lot of "wait how do certs work?" I'm going to skip ahead to the one that feels like the most reasonable to share as I am probably not the only person who got their dick caught in this fuck machine.

The Terraform file:

resource "tls_private_key" "root" {
  algorithm   = "ECDSA"
  ecdsa_curve = "P521"
}

resource "tls_self_signed_cert" "root" {
  key_algorithm   = "ECDSA"
  private_key_pem = "${tls_private_key.root.private_key_pem}"

  validity_period_hours = 26280
  early_renewal_hours   = 8760

  is_ca_certificate = true

  allowed_uses = ["cert_signing", "key_encipherment", "server_auth", "client_auth"]

  subject {
    common_name = "Autogenerated Cert"
  }
}

resource "tls_private_key" "etcd" {
  algorithm   = "ECDSA"
  ecdsa_curve = "P521"
}

resource "tls_cert_request" "etcd" {
  key_algorithm   = "${tls_private_key.etcd.algorithm}"
  private_key_pem = "${tls_private_key.etcd.private_key_pem}"

  subject {
    common_name = "${var.etcd_dns}"
  }

  dns_names = ["${var.etcd_dns}", "localhost"]
}

resource "tls_locally_signed_cert" "etcd" {
  cert_request_pem = "${tls_cert_request.etcd.cert_request_pem}"

  ca_key_algorithm   = "${tls_private_key.root.algorithm}"
  ca_private_key_pem = "${tls_private_key.root.private_key_pem}"
  ca_cert_pem        = "${tls_self_signed_cert.root.cert_pem}"
  validity_period_hours = 17520
  early_renewal_hours   = 8760

  allowed_uses = ["server_auth", "client_auth"]
}

data "template_file" "config" {
  template = "${file("config.tpl")}"

  vars {
    ca_pem_path   = "/var/lib/etcd/certs/ca.pem"
    ca_pem        = "${jsonencode(tls_self_signed_cert.root.cert_pem)}"

    etcd_key_path = "/var/lib/etcd/certs/etcd.key"
    etcd_key      = "${jsonencode(tls_private_key.etcd.private_key_pem)}"

    etcd_pem_path = "/var/lib/etcd/certs/etcd.pem"
    etcd_pem      = "${jsonencode(tls_locally_signed_cert.etcd.cert_pem)}"
  }
}

resource "local_file" "config" {
  content  = "${data.template_file.config.rendered}"
  filename = "config.yaml"
}

resource "local_file" "ca" {
  content  = "${tls_self_signed_cert.root.cert_pem}"
  filename = "ca.pem"
}

resource "local_file" "key" {
  content  = "${tls_locally_signed_cert.etcd.cert_pem}"
  filename = "etcd.pem"
}

resource "local_file" "pem" {
  content  = "${tls_private_key.etcd.private_key_pem}"
  filename = "etcd.key"
}

resource "null_resource" "ct_render" {
  depends_on = ["local_file.config"]
  provisioner "local-exec" {
    command = "ct --platform=digitalocean < config.yaml > config.json"
  }
}

provider "digitalocean" {
  token = "${var.do_token}"
}


resource "digitalocean_droplet" "etcd" {
  depends_on = ["null_resource.ct_render"]

  ssh_keys           = ["${var.do_ssh_key}"]
  image              = "${var.do_image}"
  region             = "${var.do_region}"
  size               = "${var.do_size}"
  private_networking = "${var.do_networking_private}"
  backups            = "${var.do_backups}"
  ipv6               = "${var.do_ipv6}"
  name               = "${var.do_name}"
  user_data          = "${file("config.json")}"
}

resource "digitalocean_domain" "etcd"  {
  name       = "${var.etcd_dns}"
  ip_address = "${digitalocean_droplet.etcd.ipv4_address}"
}

resource "digitalocean_record" "etcd_dns" {
  domain = "elijahcaine.me"
  type   = "A"
  name   = "${var.etcd_name}"
  value  = "${digitalocean_droplet.etcd.ipv4_address}"
  ttl    = "60"
}

The output:

$ etcdctl --ca-file ca.pem --key-file etcd.key --cert-file etcd.pem --endpoints https://$PUBLIC_DNS_NAME:2379 cluster-health
failed to check the health of member 2d6aa874eef6cb44 on https://$PUBLIC_IPV4:2379: Get https://$PUBLIC_IPV4:2379/health: x509: cannot validate certificate for $PUBLIC_IPV4 because it doesn't contain any IP SANs
member 2d6aa874eef6cb44 is unreachable: [https://$PUBLIC_IPV4:2379] are all unreachable
cluster is unhealthy

Dafauq?

So I made an issue

And discovered this a few hours later:

$ curl --cert etcd.pem --key etcd.key --cacert ca.pem https://$PUBLIC_DNS_NAME:2379/v2/members
{"members":[{"id":"2d6aa874eef6cb44","name":"$HOSTNAME","peerURLs":["https://$PRIVATE_IPV4:2380"],"clientURLs":["https://$PUBLIC_IPV4:2379"]}]}

Wait, dafauq? I thought the cert -- but over there it failed -- wait no. Nooooooo.

I realized I might be using a version of etcd which introduced a bug, so I bumped down to 3.1.6. Same error.

THEN I realized I was using the v2 api in etcdctl

$ etcdctl --version
etcdctl version: 3.0.6
API version: 2

And so I tried setting the API to v3 and of course it just fucking worked.

$ ETCDCTL_API=3 etcdctl --cacert ca.pem --key etcd.key --cert etcd.pem --endpoints https://$PUBLIC_DNS_NAME:2379 member list
a490e764e3257252, started, pipewrench, https://PRIVATE_IPV4:2380, https://PUBLIC_IPV4:2379

Goddamit. There goes a week of my life I'll never get back.

Two nodes are better than one
