Anybody that's made a website probably remembers their first time:</p> <ol class="arabic simple"> <li>Open up a text editor.</li> <li>Type something like the following in:</li> </ol> <pre class="code html literal-block"> <span class="p">&lt;</span><span class="nt">h1</span><span class="p">&gt;</span>Hello world!<span class="p">&lt;/</span><span class="nt">h1</span><span class="p">&gt;</span> <span class="p">&lt;</span><span class="nt">p</span><span class="p">&gt;</span>This is so cool!<span class="p">&lt;/</span><span class="nt">h1</span><span class="p">&gt;</span> </pre> <ol class="arabic simple" start="3"> <li><tt class="docutils literal">Save As index.html</tt>.</li> <li>Right click and open the file in your browser.</li> <li>Marvel at what you've created and hack away at it all night long.</li> </ol> <p>And there you go! You've got a website made and ready to roll. It's not on a server, and you can't tell your friends to go to it from <em>their</em> computer -- but those are just technicalities. You can still celebrate doing a thing <em>like a boss</em>.</p> <p>Unfortunately, when you test your website by viewing local files you're missing out on a lot of advantages and quirks that you get when you host a website on a <em>real server</em> using something like <a class="reference external" href="https://en.wikipedia.org/wiki/Web_server">Apache or Nginx</a>.</p> <p>For instance, when you run check your website by clicking through <tt class="docutils literal"><span class="pre">file///home/username/project/files.html</span></tt> all of your hyperlinks that should point to <tt class="docutils literal"><span class="pre">http://mywebsite.ext/somepage/</span></tt> will take you to <tt class="docutils literal"><span class="pre">file:////somepage</span></tt> when they should take you to <tt class="docutils literal"><span class="pre">file:///home/username/projects/myawesomesite/somepage/index.html</span></tt>.</p> <div class="section" id="run-a-development-server-it-s-easy"> <h4>Run A Development Server (it's easy!)</h4> <div class="section" id="the-short-answer"> <h5>The Short Answer</h5> <p><strong>Use Python!</strong></p> <pre class="code text literal-block"> $ python2 -m SimpleHTTPServer Serving HTTP on 0.0.0.0 port 8000 ... </pre> </div> <div class="section" id="the-longer-answer"> <h5>The Longer Answer</h5> <p>You can run your own local webserver to serve files <em>locally</em>. This gets you all of the developmental advantages of running a webserver without having to rent or run a server in the cloud. The best part is that it's very easy to run a development web server.</p> <p>Here's how:</p> <ol class="arabic simple"> <li><a class="reference external" href="https://python.org/downloads/">Install Python</a></li> <li>Navigate to <tt class="docutils literal"><span class="pre">your-awesome-website</span></tt> directory/folder.</li> <li>Run <tt class="docutils literal">python <span class="pre">-m</span> SimpleHTTPServer</tt> if you installed <tt class="docutils literal">python2</tt> or run <tt class="docutils literal">python <span class="pre">-m</span> http.server</tt> if you installed <tt class="docutils literal">python3</tt>.</li> </ol> <div class="note"> <p class="first admonition-title">Note</p> <p class="last">These instructions assume you are doing things in a Unix environment (OSX or Linux) thought the commandline.
